MudOS 是一个 LPmud 服务器(驱动), 最初作为一个分布式的增强版 LPmud 3.x 驱动. 
它兼容方面比较差, 但许多新 mudlibs 是可用的.

小版本的 MudOS (那些次要版本号后, 如 v20.26 十进制) 通常是正式发布, 没有安装
说明、使用说明、文档或其他支持材料(因为这其他材料的大小相当大). MudOS 版本与
版本号只有一个版本(如v21)会发布时连同源代码及文档

小版本是为那些已经熟悉安装和运行 MudOS 的人准备. 将修复 bug 和引入新特性.

α 和 β 版本的未来版本只提供补丁的形式.

一般来说, 只有那些有兴趣测试的同学们采用 α/β 版本。

当决定升级时, 一定要首先阅读处于每一级目录下的 README 来了解最新的源代码的改
动, 对于 α 和 β 版本, 建议也看一下更新日志. 特别是标着 “兼容性” 的地方, 因为
这些描述了驱动程序修改, 可能导致预先存在的 mudlibs(如 Lil) 产生问题.

注意:

官方支持站点 http://www.mudos.org/ 已经不可用了, 可能已经停止了该项目.

如果你认为你已经找到了一个错误没有在 MudOS 错误列表中, 暂无报告之处.

如果驱动输出当前操作在 socket 上不支持, 尝试在编译时使用 cc 替换 gcc (如果你
是用 gcc 的话). 另外, 尝试取消在 Makefile 文件中对 NEED_OLD_CPP 的注释, 然后
执行 'make clean' 和 'make'.

子目录:

amiga/          包含 amiga 支持文件, 加密包还提供了系统缺失的 DES crypt()
include/        有用的 #defines 用于访问 mudlib (用于各种 efuns)
packages/       efun 包和用户 efuns 存放处

本代码自 2010 年前的 MudOS v22.2b14 with mysql linux 源码开始.
